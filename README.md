# IT Kinderschule — сайт школи робототехніки та програмування

Сучасний двомовний (🇺🇦 UA / 🇩🇪 DE) сайт для дитячої ІТ-школи в Гамбурзі з:

- 🎨 «лагідним», але насиченим інтерфейсом (світла/темна тема, анімації, лайтбокс, маркі-стрічка);
- 📝 зручним **багатокроковим майстром заявки** на курси;
- 📬 надсиланням заявок **на пошту** (адмін + автовідповідь батькам) через Nodemailer;
- 🔐 **адмін-панеллю** зі статистикою, фільтрами, статусами, нотатками та експортом у CSV;
- ✏️ **редагуванням контенту прямо з адмінки** (CMS): головний банер (Hero), секція
  «Про нас», курси, викладачі, галерея, відгуки, FAQ, соцмережі та контакти —
  зміни застосовуються миттєво, **без перезапуску сервера**;
- 🖼️ **завантаженням зображень з адмінки** (фото галереї, фото викладачів, фон Hero,
  фото «Про нас») — без доступу до файлів сервера;
- ✅ вибором **кількох курсів** одночасно у формі заявки;
- 💾 збереженням заявок у простій файловій базі (без БД-серверів).

## 🚀 Швидкий старт

```bash
cd server
npm install
cp .env.example .env      # на Windows: copy .env.example .env
# відредагуйте .env (пошта, пароль адміна) — див. нижче
npm start
```

Відкрийте:
- Сайт — http://localhost:3000
- Адмін-панель — http://localhost:3000/admin

> Сайт працює **навіть без налаштованої пошти** — заявки все одно зберігаються
> й видно в адмін-панелі. Пошта — це додаткове сповіщення.

## ⚙️ Налаштування `.env`

```ini
PORT=3000

# Куди приходять заявки
ADMIN_EMAIL=radarix75@gmail.com

# SMTP-акаунт для надсилання (Gmail)
SMTP_HOST=smtp.gmail.com
SMTP_PORT=587
SMTP_USER=ваш.акаунт@gmail.com
SMTP_PASS=app-password-16-символів
MAIL_FROM_NAME=IT Kinderschule
SEND_CONFIRMATION=true       # автолист батькам

# Вхід в адмін-панель
ADMIN_LOGIN=admin
ADMIN_PASSWORD=надійний-пароль
AUTH_SECRET=довгий-випадковий-рядок
```

### Як отримати Gmail App Password
1. Увімкніть двофакторну автентифікацію: https://myaccount.google.com/security
2. Створіть пароль додатка: https://myaccount.google.com/apppasswords
3. Скопіюйте 16 символів у `SMTP_PASS`.

Можна використати будь-який інший SMTP (Outlook, Web.de, власний) — змініть `SMTP_HOST/PORT/USER/PASS`.

## 📬 Варіанти подачі заявок на пошту

Реалізовано **варіант 1** (рекомендований). Інші — на випадок зміни хостингу:

| # | Підхід | Плюси | Мінуси |
|---|--------|-------|--------|
| **1** | **Node + Nodemailer (цей проєкт)** | Пошта **і** адмін-панель з історією, повний контроль, безкоштовно | Потрібен хостинг із Node |
| 2 | Web3Forms / Formspree | Працює на статиці (GitHub Pages), 0 коду на сервері | Немає власної адмінки/історії |
| 3 | SendGrid / Resend API | Найкраща доставлюваність на великих обсягах | Реєстрація, ключі, ліміти |

Щоб перейти на **варіант 2** (статика без сервера) — у формі замінити `fetch('/api/applications')`
на endpoint Web3Forms; адмінка тоді недоступна.

## 🔌 API

| Метод | Шлях | Доступ | Опис |
|-------|------|--------|------|
| POST | `/api/applications` | публічно | нова заявка (+ лист) |
| POST | `/api/newsletter` | публічно | підписка на розсилку |
| POST | `/api/admin/login` | публічно | вхід, повертає токен |
| GET | `/api/admin/applications` | 🔐 | список заявок |
| GET | `/api/admin/stats` | 🔐 | статистика |
| PATCH | `/api/admin/applications/:id` | 🔐 | статус / нотатка |
| DELETE | `/api/admin/applications/:id` | 🔐 | видалити |
| GET | `/api/admin/export.csv` | 🔐 | експорт у CSV |
| GET | `/api/content` | публічно | контент сайту (курси, контакти…) |
| GET/PUT | `/api/admin/content` | 🔐 | читати / зберігати контент (CMS) |
| POST | `/api/admin/upload` | 🔐 | завантажити зображення (JPG/PNG/WebP/GIF, ≤8 МБ) |
| GET | `/api/admin/uploads` | 🔐 | список завантажених зображень |
| GET | `/api/admin/subscribers` | 🔐 | список підписників розсилки |
| GET | `/api/admin/subscribers/export.csv` | 🔐 | експорт підписників у CSV |
| DELETE | `/api/admin/subscribers/:id` | 🔐 | видалити підписника |

Захист: rate-limit, honeypot проти ботів, валідація на сервері, токени з терміном дії 12 год.

## 🌍 Деплой

📄 **Повна покрокова інструкція — у файлі [DEPLOY.md](DEPLOY.md)** (Render, Railway,
VPS, Docker/Fly.io + як заповнити змінні та зберегти дані на постійному диску).

Коротко:
- Хостинг запускає `npm install` → `npm start` з кореня проєкту (є кореневий `package.json`).
- Усі налаштування — через змінні середовища (див. `server/.env.example` і DEPLOY.md).
- Health-check: `GET /healthz`.
- ⚠️ Підключіть постійний диск і задайте `DATA_DIR` / `UPLOAD_DIR`, щоб заявки,
  контент і фото не зникали після передеплою.

## ✏️ Як редагувати контент

**Найпростіше — через адмін-панель** (`/admin` → вкладка «Контент»):
головний банер (Hero), «Про нас», курси, викладачі, галерея, відгуки, FAQ,
соцмережі та контакти. Натисніть «Зберегти та опублікувати» — і зміни одразу
на сайті. Новий курс автоматично з'являється й у формі заявки. Щоб приховати
іконку соцмережі — залиште поле порожнім. У заголовку Hero виділіть слово
зірочками (`*майбутнє*`) — воно стане кольоровим. Фото (галерея, викладачі,
Hero, «Про нас») завантажуються кнопкою прямо в адмінці й лягають у
`public/images/uploads/`. Дані зберігаються у `data/content.json`.

Через код (за бажанням):
- Початковий контент (засів) — `server/content.js` (`DEFAULT_CONTENT`).
- Резервні дані фронтенду, якщо API недоступне — `public/js/data.js`.
- Переклади інтерфейсу (UA/DE) — `public/js/i18n.js`.
- Кольори/стиль — змінні `:root` у `public/css/style.css`.
- Фото — папка `public/images/`.

> Щоб скинути контент до заводського: зупиніть сервер, видаліть
> `data/content.json` і запустіть знову.
